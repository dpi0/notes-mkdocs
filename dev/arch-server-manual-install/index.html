    Arch Server Manual Install

    Note

    if installing via KVM

    1. configure your nftables
    2. check ip -c a and ping google.com as KVM routes traffic through me already
    3. set a root passwd: passwd
    4. ssh root@ip-from-above enter password and copy paste
    cat /sys/firmware/efi/fw_platform_size

setfont ter-132n
loadkeys us

iwctl -P "password" station wlan0 connect "SSID"
ping archlinux.org

timedatectl set-timezone Asia/Kolkata
timedatectl

lsblk -f
cfdisk /dev/sdX or /dev/vdX
    • /efi EFI System fat32 = 200M (.efi files, and systemd-boot conf)
    • /boot Linux extended boot fat32 = 500M (kernel, initramfs and ucode)
    • / Linux root (x86-64) ext4 = 65-75G (on servers images and volumes take up a lot of space, and this wiil also house swapfile 4G)
    • /home Linux home ext4 = rest of it

    partitioning¶

    mkfs.fat -F32 /dev/sdX-efi

mkfs.fat -F32 /dev/sdX-boot
mkfs.ext4 /dev/sdX-root
mkfs.ext4 /dev/sdX-home

mount /dev/sdX-root /mnt
mkdir /mnt/efi /mnt/boot /mnt/efi /mnt/home
mount /dev/sdX-efi /mnt/efi
mount /dev/sdX-boot /mnt/boot
mount /dev/sdX-home /mnt/home

    setup systemd-boot¶

    sudo bootctl --esp-path=/efi --boot-path=/boot install

sudo tee -a /efi/loader/loader.conf << EOF
timeout 2
default arch.conf
console-mode max
editor no
EOF

UUID=$(blkid -s UUID -o value /dev/sdaX-root)

sudo tee -a /boot/loader/entries/arch.conf << EOF
title Arch Server
linux /vmlinuz-linux-lts
initrd /initramfs-linux-lts.img
options root=UUID=$UUID rw
EOF

    exit

    further recommendations for boosting perf on low ram¶

    https://wiki.archlinux.org/title/Zram#Optimizing_swap_on_zram

    sudo tee -a /etc/sysctl.d/99-vm-zram-parameters.conf << EOF
vm.swappiness = 180
vm.watermark_boost_factor = 0
vm.watermark_scale_factor = 125
vm.page-cluster = 0
EOF

sudo sysctl --system
    • vm.swappiness = 180: This setting controls how aggressively the kernel will swap memory pages. A high value like 180 makes the kernel more likely to use zram swap over physical RAM when it gets filled up.
    • vm.watermark_boost_factor = 0: This setting disables the boosting of watermarks, which are thresholds for free memory.
    • vm.watermark_scale_factor = 125: This increases the watermarks by a factor of 1.25, helping to manage memory more aggressively.
    • vm.page-cluster = 0: This setting determines the number of pages the kernel will read or write in a single swap operation. Setting it to 0 optimizes the system for the smaller and faster swap operations that zram can handle.

    harden ssh¶

    first setup ssh keys - on host¶

    ssh-keygen -t ed25519 -f ~/.ssh/home-server -C "home server"
sudo chmod 600 ~/.ssh/home-server

ssh-copy-id -i ~/.ssh/home-server.pub dpi0@server

    now on server¶

    sudo tee -a /etc/ssh/sshd_config <<EOF
AuthorizedKeysFile      .ssh/authorized_keys

Subsystem       sftp    /usr/lib/ssh/sftp-server

Port 3000

AllowUsers dpi0 bruh

PermitRootLogin no

PasswordAuthentication no

PermitEmptyPasswords no

ChallengeResponseAuthentication no

X11Forwarding no

AllowTcpForwarding no

AllowAgentForwarding no

Match user dpi0
        PubkeyAuthentication yes

Match User bruh
        PasswordAuthentication yes
        PubkeyAuthentication no
EOF

sudo systemctl restart sshd

ssh -p 3000 -i ~/.ssh/home-server dpi0@server (only pubkey for dpi0)

    auto mount drives/partitions on boot¶

    https://wiki.archlinux.org/title/Fstab

    UUID=abc /     ext4   defaults  0      1
UUID=xyz /home ext4   defaults  0      2

    is checked by the dump(8) utility. This field is usually set to 0, which disables the check.

    sets the order for file system checks at boot time; see fsck(8). For the root device it should be 1. For other partitions it should be 2, or 0 to disable checking.

    configure firewall¶

    just chicken out and use ufw for sanity sake¶

    sudo pacman -S ufw

sudo ufw enable
sudo systemctl enable ufw
sudo ufw limit 3000/tcp # ssh
sudo ufw limit 2222/tcp # ssh gitea
sudo ufw allow 2049/tcp # nfs
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw default allow outgoing
sudo ufw default deny incoming

sudo ufw status verbose
sudo ufw reload

sudo ufw status numbered
sudo ufw delete <number>
