    1. Arr Stack Home Server

    Note

    follow order

    1.1 1. radarr¶

    • films (gets stuff done with prowlarr and bazarr)
    • use the compose file to spin up
    • https://films.dpi0.cloud settings > media management > root folders. add /movies (no refresh needed)
    • settings > general > security > API Key. copy that key (for jellyseerr)

    1.2 2. sonarr¶

    • tv (gets stuff done with prowlarr and bazarr)
    • use the compose file to spin up
    • https://tv.dpi0.cloud settings > media management > root folders. add /tv (no refresh needed)
    • settings > general > security > API Key. copy that key (for jellyseerr)

    1.2.0.1 radarr rename files and directories¶

    first set the movie rename formats https://trash-guides.info/Radarr/Radarr-recommended-naming-scheme/#standard-folder

    movie

    {Movie CleanTitle} {(Release Year)} - {Edition Tags }{[Custom Formats]}{[Quality Full]}{[MediaInfo 3D]}{[MediaInfo VideoDynamicRangeType]}{[Mediainfo AudioCodec}{ Mediainfo AudioChannels]}{[Mediainfo VideoCodec]}{-Release Group}

    folder

    {Movie CleanTitle} ({Release Year})

    1.2.0.2 first import dirs/library containg movies directly under root dir¶

    • go to movies tab
    • import root dir containing movies

    1.2.0.3 rename dir names¶

    • go to movies tab
    • navbar -> edit movies -> select only the movies/dirs from the single library
    • footer -> edit -> root folder -> SELECT ONLY THE DIR FROM WHICH THE CURRENT MOVIES ARE SELECTED (otherwise the move will mess things up bad)

    1.2.0.4 rename movie video file names¶

    • go to movies tab
    • navbar -> edit movies -> select only the movies/dirs from the single library
    • footer -> rename files -> organize

    1.3 3. jellyseerr¶

    • request movies/tv (sends requests to radarr/sonarr)
    • use the compose file to spin up
    • https://watch.dpi0.cloud add the jellyfin server (easy) and scan the library
    • continue (make sure radarr has been configured as above before adding it here)
    • "Add New Radarr Server". check default. server name (anything) "radarr". hostname (service name) "radarr". api key paste. then TEST
    • now set quality profile "1080p" and root folder "/movies"
    • add server
    • "Add New Sonarr Server". check default. server name (anything) "sonarr". hostname (service name) "sonarr". api key paste. then TEST
    • now set quality profile "1080p" and root folder "/tv" and language profile "deprecated" fsr?!. same for anime (if needed)
    • check season folders
    • add server
    • for testing "request" a movie like "dogville" (this'll send the request to radarr)

    1.5.1 back to radarr...¶

    • now add a download client. settings > download clients. add qb. host "qb" (the container name). port "8090" (internally exposed port). username and password of qb. test and move on.
    • delete that "dogville" as even refreshing doesn't work. start a new.

    1.6 6. bazarr¶

    • subtitles
    • use the compose file to spin up
    • https://sub.dpi0.cloud and follow the https://wiki.bazarr.media/Getting-Started/Setup-Guide/

    1.7 final settings¶

    me using qb

    • tag private stuff with my-films and my-tv

    anyone else using qb

    • tag movies with radarr and tv-sonarr

    the flow for manual import for a normal user should be:

    1. search the movie/tv on public trackers 1337x, rarbgdump etc
    2. add the movie/tv to jellyseer

    not the other way round! (easily possible but leads to confusion on edge cases)

    1.7.0.1 jellyseer¶

    all requests go to a single public directory in /hdd/Media/Films/Public or /hdd/Media/TV/Public

    there's no around this of achieving user privacy of directing movies to seperate directories. not without using multiple instances of the entire stack!

    1.7.0.2 radarr¶

    radarr receives requests from jellyseer, adds them to library. if an indexer is configured it uses that for the movie. if not then waits for a manual movie addition by qb.

    radarr only cares about the tags present on the download clients. if the tags match, the movie will show up.

    1.7.0.3 qbitorrent¶

    there is no way to have multiple users on the same qb instance. so to achieve basic privacy when adding manually, using seperate instances.

    you can achieve seperate downloads for manual imports, using tags. but again no way to achieve privacy on the Media directory. not without using seperate instances of the arr stack.

    https://trash-guides.info/Downloaders/qBittorrent/Basic-Setup/

    Your download client ONLY downloads to your download folder/location.

    And you tell Radarr where you want your clean media library

    Starr Apps import from your download location (copy/move/hardlink) to your media folder/library

    Plex, Emby, JellyFin, or Kodi should ONLY have access to your media folder/library

    Your Download and Media Library should be NEVER the same locations

    • yes, Pre-allocated disk space (useful for HDDs) https://rimgo.in.projectsegfau.lt/a/kBQfkhS
    • use all these settings in saving management except for default torrent management keep that to manual (for my current use case) https://trash-guides.info/Downloaders/qBittorrent/Basic-Setup/#saving-management
    • use all these in connection https://trash-guides.info/Downloaders/qBittorrent/Basic-Setup/#listening-portjk

    1.7.0.4 radarr¶

    https://trash-guides.info/File-and-Folder-Structure/Examples/

    • set root folders and check hardlinks (if not already)

    • add download client: host=qb, port=8090, username, password, add tag=radarr

    2. add movie to qb (wait for it to finish, radarr will then auto add the corrected movie to Media :>)
