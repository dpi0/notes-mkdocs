    1. DNS Records

    DNS protocol can contain multiple pieces of data about a domain, each called records

    1.1 A record¶

    hold IPv4 addresses dog A google.com --> A google.com. 2m31s 142.250.194.46

    dog A dpi0.github.io
A dpi0.github.io. 1h00m00s   185.199.111.153
A dpi0.github.io. 1h00m00s   185.199.110.153
A dpi0.github.io. 1h00m00s   185.199.108.153
A dpi0.github.io. 1h00m00s   185.199.109.153

    1.2 AAAA record¶

    hold IPv6 addresses dog AAAA google.com --> AAAA google.com. 2m08s 2404:6800:4002:821::200e

    CAA record certification authority authorisation record is used to limit the Certificate Authorities that are allowed to generate certificates for a given domain.

    dog CAA google.com
CAA google.com. 12h00m00s   "issue" "pki.goog" (non-critical)

dog CAA hostinger.com
SOA hostinger.com. 1h00m00s A "any1.hostinger.com." "dns.hostinger.com." 2021102522 3h00m00s 1h00m00s 7d0h00m00s 1h00m00s

SOA lookup.dog. 5m00s A "ns1.dnsimple.com." "admin.dnsimple.com." 1564273563 1d0h00m00s 2h00m00s 7d0h00m00s 5m00s

CAA dpi0.github.io. 59m58s   "issuewild" "digicert.com" (non-critical)

    1.3 SOA (start of authority) record¶

    contains information about the DNS zone i.e., hierarchy of domains that makes up the Domain Name System

    SOA records are commonly seen when querying a domain that does not exist — even when the query is not asking for SOA records specifically.

    It contains the following fields, in order:
    1, The name of the primary nameserver for this zone. 2. The e-mail address of the party responsible for the zone. 3. (This is encoded where the first “.” becomes an “@”.) 4. The internal serial number for this domain. 5. The duration after which the zone should be refreshed. 6. The duration before a failed refresh operation should be retried. 7. The duration after which this zone should no longer be considered authoritative. 8. The TTL for how long to cache the non-existence of domains inside this zone.

    dog SOA google.com
SOA google.com. 55s A "ns1.google.com." "dns-admin.google.com." 639491960 15m00s 15m00s 30m00s 1m00s

    1.4 SIG (Signature) record¶

    used in encryption protocols.

    1.5 SRV (service) record¶

    enhancement of the A and AAAA records, used to return not just addresses, but address–port combinations, again everyone gave SOA

    dog SRV srv-example.lookup.dog
SRV srv-example.lookup.dog. 1h00m00s   10 20 "dns.lookup.dog.":5000
  A dns.lookup.dog.           53m49s + 138.68.117.94

    1.6 CNAME record¶

    canonical name it aliases one record to another, works at the domain-name level, rather than the website level if hosted at “example.com”, and you want to also make it available at “example.net”, you could CNAME one domain to another. but if “example.com/mysite”, then you cannot make the alias — there’s no way to add the “/mysite” path.

    dog CNAME google.com
SOA google.com. 52s A "ns1.google.com." "dns-admin.google.com." 639491960 15m00s 15m00s 30m00s 1m00s

dog CNAME dpi0.github.io
SOA github.io. 15m00s A "ns-1622.awsdns-10.co.uk." "awsdns-hostmaster.amazon.com." 1 2h00m00s 15m00s 14d0h00m00s 1d0h00m00s

    1.7 HINFO (host information) record¶

    contains the CPU and operating system descriptions, containing the CPU field, and the second containing the OS field

dog HINFO dns.lookup.dog
SOA lookup.dog. 5m00s A "ns1.dnsimple.com." "admin.dnsimple.com." 1564273563 1d0h00m00s 2h00m00s 7d0h00m00s 5m00s

    1.11 OPT (options) record¶

    contains extra fields that were added to DNS later

    dog google.com --edns show
  A google.com. 2m18s   142.250.194.46
OPT                   + 512 0 0 0 []

dog hostinger.com --edns show
  A hostinger.com. 3m30s   104.16.146.108
  A hostinger.com. 3m30s   104.16.147.108
OPT                      + 512 0 0 0 []

    1.12 PTR (pointer) record¶

    holds a pointer to a canonical name. This is most often used for reverse DNS lookups. everyone i tested gave SOA

    dog PTR 1.0.0.127.in-addr.arpa
PTR 1.0.0.127.in-addr.arpa. 0s   "localhost."

    instead of finding the IP for a domain name, we are asking for the hostname of an IP — in this case, the IP address “127.0.0.1” i.e localhost

    1.13 TXT (text) record¶

    holds arbitrary ASCII text, Internet’s infrastructure, TXT records are used by applications that run atop it, as a way to make short pieces of data available to the public

    use case is to prove that a domain belongs to a particular party, as only somebody in control of the domain can set records on it
