    1.3 hibernation/suspend - power save¶

    https://wiki.archlinux.org/title/Power_management/Suspend_and_hibernate#Hibernation

    • sudo systemctl suspend: sleep will save the system state to RAM.
    • sudo systemctl hybrid-sleep: suspends the system both to RAM and disk, so a complete power loss does not result in lost data.
    • sudo systemctl suspend-then-hibernate: initially suspends the system to RAM as long as possible, then wakes it with an RTC alarm and hibernates. Saves the machine's state into swap space, but does not power off the machine. Instead, it invokes the default suspend. Therefore, if the battery is not depleted, the system can resume instantly.
    • sudo systemctl hibernate: Saves the machine's state into swap space and completely powers off the machine

    1.3.1 create swapfile¶

sudo mkswap -U clear --size 4G --file /swapfile

# to increase/decrease swapfile size
sudo swapoff /swapfile

    1.3.3 for a swapfile, get the file offset and configure bootloader¶

    sudo filefrag -v /swapfile | awk '$1=="0:" {print substr($4, 1, length($4)-2)}'

sudo vim /boot/loader/entries/arch.conf
# UUID is of the partition where /swapfile is located
# resume_offset is taken from the above command
---
options root=UUID=20bb085b-860b-412d-b65d-0b9c807a449b rw resume=UUID=20bb085b-860b-412d-b65d-0b9c807a449b resume_offset=13467648
---

    then try: sudo systemctl hibernate and see it resumes the state or not (haven't tested it on home-server as pointless)

    1.10.0.1 Configuring SSH Keys with GitHub¶

    1. Create a key pair:
      ssh-keygen -t ed25519 -f ~/.ssh/github-key -C "ssh key for github"
    2. Skip the passphrase for now.
    3. Add the PUBLIC KEY (github-key.pub) to your GitHub account.
    4. Create a directory with the same name as the GitHub repo (e.g., test-github-dir).
    5. Test the connection:
      ssh -T -i ~/.ssh/github-key git@github.com
    6. For every shell session, run:
      eval "$(ssh-agent -s)"
ssh-add ~/.ssh/github-key
    7. configure global name and email
    git config --global user.name "dpi0"
git config --global user.email "dpi0.dev@proton.me"

cat ~/.gitconfig

    1.12 immich¶

    • download the .env file from here https://immich.app/docs/install/docker-compose
    • in it change, upload location to /hdd/Immich/upload and database immich_db

    Note

    the core issue is that: you can not control the ownership/permissions of a file/dir on NTFS drives using ntfs-3g. you simply can not — it is not designed to do that.

    so doing this in .env

    DB_DATA_LOCATION=/hdd/Immich/postgres

    won't work. as immich fsr needs to set 999:root for this directory, as ntfs-3g is unable to set 999

  -v $LOCATION:/mnt \
  matrixdotorg/dendrite-monolith:latest \
  -private-key /mnt/matrix_key.pem

docker run --rm --entrypoint="/bin/sh" \
  -v $LOCATION:/mnt \
  matrixdotorg/dendrite-monolith:latest \
  -c "/usr/bin/generate-config \
    -dir /var/dendrite/ \
    -db postgres://dendrite:$DB_PASSWORD@postgres/dendrite?sslmode=disable \
    -server $SERVER_NAME > /mnt/dendrite.yaml"

sed -i "s/POSTGRES_PASSWORD:.*/POSTGRES_PASSWORD: $DB_PASSWORD/" docker-compose.yaml

unset DB_PASSWORD
unset LOCATION
unset SERVER_NAME

docker compose down; docker compose up -d

    #!/bin/bash

# Get a list of all container IDs
containers=$(docker ps -aq)

# Iterate over each container
for container in $containers; do
    echo "Container ID: $container"
    docker inspect $container | jq -r '.[0].Mounts[] | select(.Type == "bind") | "\(.Source) -> \(.Destination)"'
    echo "----------------------------------------"
done

    Tip

    get the list of only the named volumes (excluding any anonymous volumes)

    docker volume ls | rg -v '^local\s+[a-f0-9]{64}$'

# only a clean list of volumes (for scripting)
docker volume ls | rg -v '^local\s+[a-f0-9]{64}$' | tail -n +2 | awk '{print $2}'

    Tip

    get the list of named volumes only with their mounted paths (again for scripting)

    #!/bin/bash

# Get the list of running containers
containers=$(docker ps --format "{{.Names}}")

# Loop through each container
for container in $containers; do
  # Inspect the container to get the mounts
  mounts=$(docker inspect -f '{{range .Mounts}}{{if eq .Type "volume"}}{{.Name}} {{.Source}} {{.Destination}}{{println}}{{end}}{{end}}' "$container")

  # Filter out anonymous volumes and empty lines
  while IFS= read -r line; do
    if [[ $line =~ ^[a-f0-9]{64} ]] || [[ -z $line ]]; then
      continue
    fi
    echo "$line"
  done <<< "$mounts"
done
